1. What is the difference between map(), filter(), forEach() and find()?

Ans: map(), filter(), foreach(), and find() are all array methods. But each of the methods works in different situations.

I. The map() method takes an array and a callback function as parameters. Then it goes through a loop to each of the elements of the array. After that, it returns the result as an array.

II. The filter() method filters and returns an array according to the condition passed by the function.

III. The forEach() method executes a function once for every element in an array. It never returns anything.

IV. The find() method works exactly as like the filter() method. But the difference is that the filter() returns an array of all the array elements which pass the condition in the function. On the hand, the find() returns only the first element which passes the condition.

2. What is the difference between local storage, sessional storage and cookies?

Ans: Local storage, session storage and cookies usually store the data of the user. But the difference is the data of local storage will not disappear after closing the browser or browser tab, the data of session storage will. The data of cookies can be set to expire at a certain time.

Also, these have a limitation in the capacity of how much data they can store. Local storage can store 10MB, session storage can 5MB and cookies can store only 4KB of data.

3. How do event loop works?

Ans: The event loop is a process that waits to be clear of the call stack to push the next function or event to execute. When the call stack is empty, the event loop checks the task queue if any function is waiting to be executed. if any, the event loop pushes it to the call stack. This is how the Event Loop works.
